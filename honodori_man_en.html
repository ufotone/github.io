honodori

FM+PSG Extended Synth — A multi-channel synthesizer that runs in the browser

honodori is an integrated synthesizer with a built-in step sequencer running on the Web Audio API. Subtractive synthesis, FM synthesis, PSG synthesis, drum machine, and PCM sampler are all combined in one screen.

02

Smartphone Controls

Pinch Zoom

Pinch in/out with two fingers to scale the entire UI (up to 10×). Zoom in for easier access to fine parameters. On desktop, the scroll wheel works the same way.

📱
Touch operation tips
All operations are single-tap toggles. Long press and double-tap are not required. Sliders can be changed continuously by dragging.

Notifications (Toast)

Instead of alert() dialogs, toast notifications appear from the bottom of the screen. Information is conveyed without interrupting your workflow.

✓ SAVED
ERROR: invalid value
⚠ bank is empty

Confirmation Dialogs

Only irreversible operations such as deletion and clearing display a custom confirmation dialog (e.g. the CLRA button). All other operations take effect immediately.

CommandDescription

Note length notation
Append a number after a note to specify its length individually. Example: c4 = quarter note C, a8 = eighth note A, g16 = sixteenth note G. Omitting the number uses the current default length set by l.

19

Save / Load

All data is saved as a single JSON file and loaded back from it. There is no auto-save and no localStorage — always save manually.

SAVE / LOAD
💾 SAVE → honodori_save.json downloaded
📂 LOAD → select .json file → all settings restored
ItemDetails
Save formatJSON text file. Filename: honodori_save.json
What is savedAll banks (steps, notes, vol, pan, tie, porta), song data, tone banks, FX settings, PCM samples (base64), BPM, GAIN.
Auto-saveNone. Closing the browser without saving loses all data.
localStorageNot used. Data is not stored in the browser.
⚠
Save before closing.
honodori does not use localStorage. All data is lost if you close or refresh the browser tab without saving.
20

Save Data Format

The saved JSON has the following top-level keys:

{
  "version": "4.2",
  "bpm": 120,
  "gain": 0.3,
  "mode": "PATTERN",
  "currentBank": 0,
  "banks": [ /* array of 16 bank objects */ ],
  "song": [ /* array of bank indices */ ],
  "tones": { /* FM/PSG tone bank presets */ },
  "fx": { /* delay/chorus/filter settings */ },
  "pcm": [ /* base64-encoded WAV data per slot */ ]
}
KeyTypeDescription
versionstringSave data format version.
bpmnumberGlobal BPM (40–300).
gainnumberMaster gain (0.0–1.0).
modestringActive mode: "PATTERN" | "SONG" | "MML".
currentBanknumberIndex of the currently selected bank (0-based).
banksarray[16]Array of bank objects. Each bank contains steps for all channels.
songarrayOrdered array of bank indices for SONG mode playback.
tonesobjectTone bank presets for FM1–FM6 and PS1–PS3.
fxobjectGlobal FX settings (delay, chorus, DJ filter).
pcmarray[32]Base64-encoded WAV data for each PCM slot. null if empty.
21

PCM Bank Save

The PCM sample bank can be exported and imported independently from pattern data. This allows reusing sample sets across different projects.

PCM BANK EXPORT / IMPORT
💾 EXPORT PCM BANK → honodori_pcm.json
📂 IMPORT PCM BANK → loads PCM slots only, other data unchanged
OperationAction
EXPORT PCM BANKSave only the PCM sample bank as honodori_pcm.json.
IMPORT PCM BANKLoad a PCM bank file. Only PCM slots are replaced; all other data (patterns, FM tones, etc.) remains unchanged.
💡
Use case
Export your drum kit once, then import it into any project. PCM bank files are separate from the main save file, making them easy to share and reuse.
