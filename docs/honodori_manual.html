02

スマートフォン操作

ピンチズーム

2本指でピンチイン／アウトすることでUI全体をスケーリングできます（最大10倍）。細かいパラメータを操作したいときはズームインすると指が当たりやすくなります。デスクトップではホイールスクロールが同等の働きをします。

📱
タッチ操作のポイント
全ての操作はタップ1回のトグルで完結します。長押し・ダブルタップは不要です。スライダーはドラッグで連続変更できます。

通知（トースト）

alert()の代わりに画面下部からトースト通知が表示されます。操作を妨げずに情報を伝えます。

✓ SAVED
ERROR: 無効な値
⚠ バンクが空です

確認ダイアログ

削除・クリアなど不可逆な操作のみ、カスタムダイアログで確認が求められます（CLRAボタンなど）。それ以外の操作は即座に反映されます。

ALGO1〜8OP1〜4の接続構造。数字が大きいほど並列度が高い。

🎵
FM音色作成のヒント
TL（Total Level）の大きいオペレータは「モジュレータ（変調源）」、小さいオペレータが「キャリア（出力源）」です。モジュレータのTLを下げるほど変調が強くなり倍音が豊かになります。アルゴリズム4（2x2並列）はベースや鐘系に向きます。

種類スロット数説明

SUB トーンバンク50スロットSUBチャンネルの音色をスロットに保存。

💡
音色切替シーケンス
シーケンサーの各チャンネルタブに「音色切替行」があります。ON にしたステップで自動的に指定トーンスロットに切り替わります。1つのパターン内で動的に音色変化をつけられます。

FX / BPM シーケンス自動化

シーケンサーのFXタブ・BPMタブでは、エフェクトパラメータとテンポをステップごとに自動化できます。

項目説明
FX シーケンス各ステップで Delay WET/FB・Chorus RATE/DEPTH/WET・DJ FILTER FREQ/RES を変化させる
BPM トラック各ステップに個別のBPMを設定（nullでグローバルBPMを使用）。段階的テンポチェンジが可能
⚠
FX・BPMシーケンスをONにしたステップでは、そのステップでの再生時にパラメータが即座に切り替わります。不意の大きな音量変化に注意してください。

セーブ / ロード

現在の全設定（トーン・シーケンス・FX・バンク・SONGなど）をJSONファイルとしてエクスポート・インポートできます。

ボタン動作
💾 SAVE全設定を .json ファイルとしてダウンロード。ブラウザの「保存」ダイアログが開く。
📂 LOADファイル選択ダイアログからJSONを選んで設定を完全復元。現在の設定は上書きされる。
💾
セーブのタイミング
ブラウザをリロードすると設定はリセットされます。作業終了前に必ずSAVEしてください。JSONファイルは外部エディタで直接編集も可能です。

メモリ管理について

ページを離れる・非表示にすると AudioContext が自動的にサスペンド／クローズされ、PCMバッファも解放されます。長時間使用で動作が重くなったと感じたらリロードしてください（事前にSAVEを忘れずに）。

セーブデータの形式（honodori.json）

💾 SAVEボタンで保存されるファイルはプレーンJSON形式です（圧縮なし）。テキストエディタで直接確認・編集できます。

💡
PCMトーンバンクはメインSAVEに含まれません。
WAVサンプルは別ファイル（.hpcmb）として個別に保存します。 詳細は次のセクション「PCM バンクの保存」を参照してください。

トップレベル構造

{
  "_version": 1,           // フォーマットバージョン（現在は常に1）
  "bpm": 120,              // グローバルBPM (40〜300)
  "masterGain": 30,        // マスターボリューム (0〜100)
  "mode": "pattern",       // 再生モード: "pattern" | "song" | "mml"
  "currentBank": 0,        // 選択中のバンク番号 (0〜15)
  "currentSeqTab": "sub",  // 選択中のシーケンサータブ
  "pcmBits": 6,            // PCMビット深度
  "song": { ... },         // SONGモードのデータ
  "banks": [ ... ],        // 16バンク分のシーケンスデータ
  "fmParams": [ ... ],     // FM1〜6の音色パラメータ
  "psgParams": [ ... ],    // PSG1〜3の音色パラメータ
  "subParams": { ... },    // SUBの音色パラメータ
  "fx": { ... },           // FXエフェクトのパラメータ
  "mix": { ... },          // ミキサー（音量・パン）
  "drumsGain": { ... },    // ドラム各パートのゲイン
  "fmToneBank": [ ... ],   // FM音色バンク（最大400スロット）
  "ssgToneBank": [ ... ],  // PSG音色バンク（50スロット）
  "subToneBank": [ ... ],  // SUB音色バンク（50スロット）
  "mml": { ... }           // MMLテキストとループ設定
}

banks（シーケンスデータ）

16要素の配列で、各バンクが以下の構造を持ちます。

banks[n] = {
  "stepLen": 16,               // ステップ数 (1〜32)
  "beatGroups": [4, 4, 4, 4],  // 拍グループ（合計 = stepLen）
  "bpmTrack": [ null, ... ],   // ステップごとのBPM（null=グローバルBPM）
  "seqData": {
    "sub":  { notes, vols, on, tie, pan },  // 各32要素の配列
    "fm1":  { notes, vols, on, tie, pan },
    ...
    "psg1": { notes, vols, on, tie, pan },
    ...
    "drums": {
      kick, snare, hat, tomhi, tommid, tomlo, openhh, closehh, pcm
      // 各32要素のboolean配列
    },
    "pcmSeq": {
      "on":   [ false, ... ],   // ステップON/OFF (32要素)
      "freq": [ null, ... ],    // 再生周波数Hz (null=デフォルト)
      "vol":  [ null, ... ],    // 音量 0〜100 (null=デフォルト)
      "slot": [ null, ... ]     // スロット番号 (null=アクティブスロット)
    },
    "fmTone":  [ { on[32], slot[32] }, ... ],  // FM1〜6 音色切替 (6要素)
    "psgTone": [ { on[32], slot[32] }, ... ],  // PSG1〜3 音色切替 (3要素)
    "subTone": { on[32], slot[32] },           // SUB 音色切替
    "fxSeq": {
      "delayWet":   { on[32], vals[32] },
      "delayFb":    { on[32], vals[32] },
      "chorusRate": { on[32], vals[32] },
      "chorusDepth":{ on[32], vals[32] },
      "chorusWet":  { on[32], vals[32] },
      "djFreq":     { on[32], vals[32] },
      "djRes":      { on[32], vals[32] }
    }
  }
}

fmParams（FM音色）

fmParams[n] = {
  "algo": 4,                          // アルゴリズム (0〜7)
  "fb":   4,                          // フィードバック (0〜7)
  "lfo": {
    "enable": false, "freq": 3,

  "ops": [                            // 4オペレータ
    { "ar":31, "dr":8, "sr":0, "rr":4,
      "sl":1, "tl":20, "ks":1, "mul":1, "dt":3 },
    ...

}

fx / mix / drumsGain

トーンバンクの個別保存

FM・PSG・SUBのトーンバンクはメインSAVEに含まれますが、それぞれ個別ファイルへの書き出し・読み込みもできます。

種類ファイル名識別子
FM トーンバンクhonodori_fm.json_type: "fmToneBank"
PSG トーンバンクhonodori_psg.json_type: "psgToneBank"
SUB トーンバンクhonodori_sub.json_type: "subToneBank"

各ファイルには _version, _app: "honodori", _type, _saved（ISO日時）のヘッダーと、スロット配列が含まれます。

PCM バンクの保存・読み込み

PCMトーンバンク（最大32スロットのWAVサンプル）はメインSAVEとは別に、専用の .hpcmb ファイルとして保存します。サンプルデータはgzip圧縮＋Base64エンコードでJSON内に埋め込まれるため、ファイル1個で全スロットを自己完結に持ち運べます。

.hpcmb ファイルの内部構造

honodori_pcm.hpcmb（テキストで見るとこうなる） 
{
  "_version": 2,
  "_app": "honodori",
  "_type": "pcmToneBank",
  "_saved": "2024-01-01T00:00:00.000Z",
  "slots": [
    {
      "slot": 0,
      "name": "kick.wav",
      "bits": 6,
      "sampleRate": 16000,
      "gain": 100,
      "wav": "H4sIAAAAAAAAA..."  // gzip圧縮済みWAVのBase64
    },
    null,                        // 空スロット
    {
      "slot": 2,
      "name": "snare",
      ...
    }
  ]
}

ボタン動作
💾 PCM SAVEロード済みの全スロットを honodori_pcm.hpcmb として保存。空スロットは null で記録される。
📂 PCM LOAD.hpcmb ファイルを選択して全スロットを復元。再生に必要なAudioBufferが即座に再構築される。
📂 .bin（MUCOM88）MUCOM88用のADPCMバイナリ（YM2608形式）を読み込み。ダイアログでサンプルレートと展開先スロットを指定できる。
スロット右下の LOADそのスロットにWAVファイルを個別ロード
BITS ボタンPCMのビット深度（6 / 8 / 12 / 16bit）。低いほどローファイなADPCM風の音になる。

MUCOM88 PCMバイナリ（.bin）の形式

YM2608 ADPCM（OKI/Yamaha 4bitニブル形式）のバイナリファイルです。

0x000〜0x3FF  ヘッダー: 32エントリ × 32バイト
  [0..15]  名前（ASCII、nullパディング）
  [16..27] 予約パラメータ
  [28..29] スタートオフセット（LE16、0x400からの相対バイト数）
  [30..31] データ長（LE16、バイト数）

0x400〜EOF  ADPCMデータ（4bitニブル連結、Highニブル優先）
読み込み時の設定説明
サンプルレート8000 / 11025 / 16000 / 22050 / 44100 Hz から選択。元音源のADPCMクロック設定に合わせる。
スタートスロットS1〜S32 から展開先を指定。ヘッダーのエントリ番号にオフセットが加算される（最大32スロットまで）。
⚠
メモリ管理
PCMスロットはAudioContextのバッファをメモリに保持します。ブラウザを長時間使用する場合や大量のサンプルをロードする場合、動作が重くなったらページをリロードしてください（事前にSAVEを忘れずに）。ページ離脱・非表示時は自動でバッファが解放されます。
